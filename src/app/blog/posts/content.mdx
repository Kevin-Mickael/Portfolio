## Avatar

Remplacez le fichier `public/images/avatar.jpeg` par votre propre image d'avatar. Elle est utilisée sur la page `/about` ainsi que pour les images open-graph générées dynamiquement.

## Favicon

Remplacez le fichier `src/app/favicon.ico` par votre propre favicon.

## Contenu

Remplacez le contenu du fichier `resources/content.js` par vos propres informations. Vous pouvez utiliser des composants personnalisés dans la plupart des cas, car les props sont généralement déclarées comme `ReactNode` au lieu de `string`, mais il est important de les importer dans le fichier.

Les imports ci-dessus vous permettent d'utiliser les composants `InlineCode` et `Link` dans le fichier de contenu.

### Détails personnels

Vos informations personnelles sont utilisées dans toute l'application pour afficher des titres, labels et images personnalisés. Les langues sont affichées sur la page `/about`, ainsi que la localisation et l'heure dans l'en-tête.

<CodeBlock
    marginBottom="16"
    highlight="2-3,7-10"
    codes={[
  {
    code:
`const person = {
    firstName: "Kevin Mickael",
    lastName: "",
    get name() {
      return "...";
    },
    role: "IT Support Engineer · Software Engineer",
    avatar: "/images/avatar.jpeg",
    location: "Madagascar/Maurice", // Affichage et config technique selon la demande utilisateur
    languages: ["Français", "Anglais"], // optionnel : Laisser vide si vous ne souhaitez pas afficher de langues
};`,
    language: "tsx",
    label: "src/app/resources/content.js"
  }
]} />

### Liens sociaux

Les liens sociaux sont affichés sur la page `/about` et dans le pied de page selon le tableau configuré dans l'objet `social`. Vous pouvez définir des icônes personnalisées pour chacun, mais n'oubliez pas de les importer dans `src/once-ui/icons.ts`. Consultez la [documentation Once UI](https://once-ui.com/docs/icons) pour plus d'informations.

<CodeBlock
    marginBottom="16"
    codes={[
  {
    code:
`const social = [

    },

      icon: "email",
      link: "mailto:lorant@once-ui.com",
    },
];`,
    language: "tsx",
    label: "src/app/resources/content.js"
  }
]} />

### Pages

Chaque page possède son propre objet qui gère le contenu personnalisé, comme les titres, sous-titres, images et données dynamiques.

<CodeBlock
    marginBottom="16"
    codes={[
  {
    code:
`const home = {
    label: "Accueil",
    title: "Portfolio de Kevin Mickael",
    description: "Site portfolio présentant mon travail en tant que IT Support Engineer · Software Engineer",
    headline: <>Créer des ponts entre design et code</>,
    subline: (
      <>
        Je suis Kevin Mickael, un IT Support Engineer · Software Engineer chez <InlineCode>Nations Unies pour le développement international</InlineCode>, où je conçois des expériences utilisateurs intuitives.
        <br /> Après le travail, je développe mes propres projets.
      </>
    ),
};`,
    language: "tsx",
    label: "src/app/resources/content.js"
  }
]} />

La propriété `label` est utilisée dans la navigation de l'en-tête. La propriété `title` est utilisée pour le titre de la page et la propriété `description` pour la description de la page. Elles sont aussi utilisées dans les balises meta et open graph.
