## Présentation

Développement d’un design system flexible et hautement personnalisable, utilisant Next.js pour le front-end et Figma pour la collaboration entre designers.

## Fonctionnalités clés

- **Bibliothèque de composants** : Création d’une bibliothèque de composants UI modulaires et réutilisables avec React et styled-components dans Next.js, en mettant l’accent sur l’accessibilité et la réactivité.
- **Thématisation et personnalisation** : Intégration d’un système de thèmes permettant de changer facilement les palettes de couleurs, la typographie et les styles de mise en page grâce aux variables CSS et aux tokens Figma.
- **Intégration Figma** : Collaboration étroite avec les designers via une bibliothèque de design partagée dans Figma, synchronisée avec la base de code pour garantir des handoffs fluides et la cohérence des tokens de design.
- **Documentation et bonnes pratiques** : Rédaction d’une documentation complète avec Storybook pour présenter les composants, les usages et les bonnes pratiques, facilitant l’adoption du design system par d’autres équipes.

## Technologies utilisées

- **Next.js** : Pour des applications React rapides et server-side.
- **Figma** : Pour la création et la gestion des assets et prototypes design.
- **Styled-Components** : Pour le style des composants React de façon modulaire et thémable.
- **Storybook** : Pour construire une bibliothèque de composants interactive et documentée.

## Défis et apprentissages

Un défi majeur a été de trouver l’équilibre entre flexibilité et cohérence du design. La solution a consisté à créer des design tokens bien définis et à établir des règles claires sur la personnalisation des composants. De plus, la mise en place de workflows de collaboration efficaces entre designers et développeurs via Figma et Git a été un apprentissage précieux qui a grandement amélioré le processus.

## Résultat

Le design system est désormais utilisé activement sur plusieurs projets, ce qui a permis d’accélérer les cycles de développement, de réduire les incohérences graphiques et d’améliorer la collaboration entre design et développement. Il constitue une base solide pour faire évoluer nos produits tout en garantissant une expérience utilisateur cohérente.
